# 🏹 Документация ArrowRust (ARS) (Устарело напишу  до 11 марта)

ArrowRust превращает написание кода в проектирование потоков данных. Все операторы препроцессора преобразуются в стандартные конструкции Rust на этапе компиляции.

## Основные операторы

### 1. Транспортер (`->`)
Передает владение данными следующему узлу.
- `data -> func` => `func(data)`
- `data -> var;` => `let var = data;`

### 2. Датчик (`<-&`)
Передает неизменяемую ссылку на данные. Оригинал остается доступным.
- `data <-& func` => `func(&data)`

### 3. Сборщик (`+>`)
Распаковывает массив или кортеж в аргументы функции.
- `[a, b] +> func` => `func(a, b)`

### 4. Репликатор (`<n>`)
Создает цикл на `n` итераций.
- Синтаксис: `data <n> { ... }`
- Доступные переменные:
  - `it`: текущие данные.
  - `idx`: индекс итерации (0..n).

### 5. Утилизатор (`-x>`)
Явное завершение времени жизни объекта (вызов `drop`).
- `-my_var>` => `drop(my_var);`

## Использование CLI

- `-f <путь>`: Обработать конкретный файл.
- `-o <путь>`: Указать имя выходного файла.
- `-fwf <путь>`: Обработать все `.ars` файлы в директории (проектный режим).
- `-b`: Скомпилировать результат в бинарный файл через `rustc`.
- `-t`: Только генерация кода (по умолчанию).
